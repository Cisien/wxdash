**RED phase — Write tests first:**

Create `tests/tst_WeatherDataModel.cpp` with a QObject test class. Key design for testability:

The WeatherDataModel constructor should accept an optional `std::function<qint64()>` elapsed-time provider (per research Open Question #1 recommendation). In production, this defaults to `QElapsedTimer::elapsed()`. In tests, inject a fake clock that returns controllable values. This avoids `QTest::qWait(31000)` in tests.

**Alternative injectable approach:** The model's `checkStaleness()` method checks elapsed time from the last update. Instead of injecting a clock into the constructor, expose a `setElapsedMsForTest(qint64 ms)` method (or accept a clock function). The simplest approach: store the last-update timestamp as a member, and have `checkStaleness()` compare against `QElapsedTimer::elapsed()` — but allow tests to call `checkStaleness()` after manually advancing time.

**Recommended approach:** Make `WeatherDataModel` accept a `std::function<qint64()>` in its constructor that returns "current elapsed ms since last restart." Default to a real `QElapsedTimer`. Tests inject a lambda that returns a controlled value.

Test methods:
- `applyIssUpdate_emitsTemperatureChanged`: Apply IssReading with temp=72.5, verify temperatureChanged signal fired with 72.5
- `applyIssUpdate_sameValue_noSignal`: Apply same temp twice, verify signal only fired once
- `applyBarUpdate_emitsPressureChanged`: Apply BarReading, verify pressureChanged signal
- `applyIndoorUpdate_emitsTempInChanged`: Apply IndoorReading, verify tempInChanged signal
- `applyUdpUpdate_overwritesWindSpeed`: Apply IssReading with windSpeed=5, then UdpReading with windSpeed=8, verify windSpeedChanged fires with 8
- `staleness_emitsSignalAfter30s`: Apply update, advance fake clock past 30000ms, trigger checkStaleness, verify sourceStaleChanged(true)
- `staleness_clearsAllValues`: After staleness triggers, verify temperature=0.0, humidity=0.0, etc.
- `staleness_clearEmitsSignals`: After staleness triggers, verify all per-field changed signals fired
- `recovery_afterStaleness`: After staleness, apply new IssReading, verify sourceStaleChanged(false)
- `noFalsePositiveAtStartup`: Create model, trigger checkStaleness immediately, verify sourceStale is false (QElapsedTimer not valid yet)
- `udpUpdate_resetsStalenessClock`: Apply UDP update, verify staleness clock resets

**GREEN phase — Implement:**

**src/models/WeatherDataModel.h:**
- QObject subclass with Q_OBJECT macro
- Q_PROPERTY declarations for all fields: temperature, humidity, dewPoint, heatIndex, windChill, windSpeed, windDir, windGust, rainRate, rainfallDaily, pressure, pressureTrend, uvIndex, solarRad, tempIn, humIn, dewPointIn, sourceStale
- Each Q_PROPERTY: READ accessor, NOTIFY signal
- Constructor accepts `QObject *parent = nullptr` and optional `std::function<qint64()> elapsedProvider = {}`
- Public slots: `applyIssUpdate(const IssReading&)`, `applyBarUpdate(const BarReading&)`, `applyIndoorUpdate(const IndoorReading&)`, `applyUdpUpdate(const UdpReading&)`, `checkStaleness()`
- Signals: per-field `*Changed(type value)` and `sourceStaleChanged(bool stale)`
- Private: m_-prefixed members for all fields, `QElapsedTimer m_lastUpdate`, `QTimer *m_stalenessTimer`, `std::function<qint64()> m_elapsedProvider`, `void clearAllValues()`, `static constexpr int kStalenessMs = 30000`

**src/models/WeatherDataModel.cpp:**
- Constructor: initialize m_stalenessTimer to fire every 5s, connected to checkStaleness(). If elapsedProvider is provided, use it; otherwise use m_lastUpdate.elapsed().
- `applyIssUpdate`: restart m_lastUpdate, clear stale if was stale, set all ISS fields with change-check pattern (`if (!qFuzzyCompare(m_field, newVal)) { m_field = newVal; emit fieldChanged(m_field); }`)
- `applyBarUpdate`: restart m_lastUpdate, set pressure fields
- `applyIndoorUpdate`: restart m_lastUpdate, set indoor fields
- `applyUdpUpdate`: restart m_lastUpdate, clear stale if was stale, set wind/rain fields (these are the fast-updating fields from UDP)
- `checkStaleness`: if m_lastUpdate is not valid, return (no false positive). Get elapsed time from m_elapsedProvider if set, otherwise m_lastUpdate.elapsed(). If > kStalenessMs and not already stale, set stale, clearAllValues, emit sourceStaleChanged(true). If <= kStalenessMs and was stale, set not stale, emit sourceStaleChanged(false).
- `clearAllValues`: set all numeric fields to 0.0/0, emit all changed signals
