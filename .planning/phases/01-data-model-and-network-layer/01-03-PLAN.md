Implement the HTTP poller per research Pattern 4.

**src/network/HttpPoller.h:**
- QObject subclass with Q_OBJECT macro
- Constructor accepts `const QUrl &url` and `QObject *parent = nullptr`
- `start()` public slot — creates QNAM and poll timer, begins polling
- Signals: `issReceived(IssReading)`, `barReceived(BarReading)`, `indoorReceived(IndoorReading)`
- Private: `QNetworkAccessManager *m_nam`, `QTimer *m_pollTimer`, `QNetworkReply *m_pendingReply`, `QUrl m_url`
- Private slot: `poll()`, `onReply()`
- Constants: `static constexpr int kPollIntervalMs = 10000` (10s)

**src/network/HttpPoller.cpp:**
- `start()`: Create QNAM (must be created on the thread the object lives on — this happens after moveToThread). Create poll timer with kPollIntervalMs interval, connect to poll(). Call poll() immediately for first fetch, then start timer.
- `poll()`: Abort previous reply if still pending (`m_pendingReply->abort(); m_pendingReply->deleteLater(); m_pendingReply = nullptr`). Create QNetworkRequest with `setTransferTimeout(5000)`. Issue `m_nam->get(req)`. Connect `finished` to `onReply`.
- `onReply()`: Get reply via `qobject_cast<QNetworkReply*>(sender())`. Check `reply->error() == QNetworkReply::NoError`. On success: call `JsonParser::parseCurrentConditions(reply->readAll())`, emit signals for each non-nullopt result. On error: silently ignore (per locked decision — next poll IS the retry). **ALWAYS** call `reply->deleteLater()`. Set `m_pendingReply = nullptr`.

**Important per locked decision (DATA-09):** No special retry logic. The 10s poll timer IS the retry mechanism. If the device is unreachable, the next poll attempt happens 10s later automatically.

Update `src/CMakeLists.txt` to add HttpPoller.cpp to wxdash_lib sources.

<task type="auto">
  <name>Task 2: Implement UdpReceiver with broadcast renewal</name>
  <files>src/network/UdpReceiver.h, src/network/UdpReceiver.cpp, src/CMakeLists.txt</files>
  <action>
Implement the UDP receiver per research Pattern 5.

**src/network/UdpReceiver.h:**
- QObject subclass with Q_OBJECT macro
- Constructor accepts `QNetworkAccessManager *nam` (shared with HttpPoller's QNAM — created in start()), `const QUrl &realtimeUrl`, `QObject *parent = nullptr`

**IMPORTANT design revision:** Since HttpPoller creates its QNAM in `start()` (after moveToThread), UdpReceiver cannot share it at construction time. Two approaches:
1. UdpReceiver creates its own QNAM in `start()` for the broadcast kick-off HTTP request
2. Main.cpp connects a signal from UdpReceiver to HttpPoller to trigger broadcast renewal

**Use approach 1 — simpler:** UdpReceiver creates its own QNetworkAccessManager in `start()`. The broadcast kick-off and renewal are simple fire-and-forget GET requests; a second QNAM on the same thread is fine.

- `start()` public slot — binds UDP socket, creates own QNAM, starts renewal timer, issues initial broadcast request
- Signal: `realtimeReceived(UdpReading)`
- Private: `QUdpSocket *m_socket`, `QNetworkAccessManager *m_nam`, `QUrl m_realtimeUrl`, `QTimer *m_renewalTimer`, `QTimer *m_healthTimer`, `QElapsedTimer m_lastPacketTimer`
- Private slots: `onReadyRead()`, `renewBroadcast()`, `checkUdpHealth()`
- Constants: `kRenewalIntervalMs = 3600 * 1000` (1h), `kHealthCheckMs = 5000` (5s), `kSilenceThresholdMs = 10000` (10s)

**src/network/UdpReceiver.cpp:**
- `start()`: Create own QNAM. Bind QUdpSocket to `QHostAddress::AnyIPv4, 22222` with `QUdpSocket::ShareAddress`. Connect `readyRead` to `onReadyRead`. Create renewal timer (fires every 3600s per locked decision), connect to `renewBroadcast`. Create health timer (fires every 5s), connect to `checkUdpHealth`. Call `renewBroadcast()` immediately to start the first broadcast session. Start timers.
- `onReadyRead()`: Loop `while (m_socket->hasPendingDatagrams())`: read datagram via `m_socket->receiveDatagram()`. Call `JsonParser::parseUdpDatagram(datagram.data())`. If result has_value, restart `m_lastPacketTimer`, emit `realtimeReceived(*result)`. Per locked decision: silently ignore malformed packets.
- `renewBroadcast()`: Create request to m_realtimeUrl (which includes `?duration=86400`). Set transfer timeout 5s. Fire-and-forget GET. Connect reply's finished signal to `reply->deleteLater()`.
- `checkUdpHealth()`: If `m_lastPacketTimer.isValid()` and elapsed > `kSilenceThresholdMs`, call `renewBroadcast()` to re-register. This handles the case where the WeatherLink Live rebooted and lost broadcast state.

The URL should be: `http://weatherlinklive.local.cisien.com/v1/real_time?duration=86400`

**Per locked decision (DATA-03):** Renew every 3600s for safety, even though 86400s was requested. Belt and suspenders.

**Per locked decision (DATA-09):** While UDP is down, health timer triggers re-registration attempts alongside normal HTTP polling. Silent recovery.

Update `src/CMakeLists.txt` to add UdpReceiver.cpp to wxdash_lib sources.

<task type="auto">
  <name>Task 3: Wire components in main.cpp with QThread</name>
  <files>src/main.cpp</files>
  <action>
Wire all components together per research Pattern 2 (worker objects on shared QThread).

**src/main.cpp:**
- Create QCoreApplication (no UI in Phase 1)
- Register metatypes: `qRegisterMetaType<IssReading>()`, `qRegisterMetaType<BarReading>()`, `qRegisterMetaType<IndoorReading>()`, `qRegisterMetaType<UdpReading>()`
- Create `WeatherDataModel` on the main thread (it's a QObject that will be consumed by the GUI thread in later phases): `auto *model = new WeatherDataModel(&app);`
- Define URLs:
  - HTTP: `QUrl("http://weatherlinklive.local.cisien.com/v1/current_conditions")`
  - UDP realtime: Build QUrl with `http://weatherlinklive.local.cisien.com/v1/real_time` and QUrlQuery `duration=86400`
- Create network thread: `auto *networkThread = new QThread(&app);`
- Create worker objects WITHOUT parents (required for moveToThread):
  - `auto *httpPoller = new HttpPoller(httpUrl);`
  - `auto *udpReceiver = new UdpReceiver(realtimeUrl);`
- Move workers to network thread: `httpPoller->moveToThread(networkThread);`, `udpReceiver->moveToThread(networkThread);`
- Connect cross-thread signals (Qt auto-detects QueuedConnection):
  - `httpPoller->issReceived` → `model->applyIssUpdate`
  - `httpPoller->barReceived` → `model->applyBarUpdate`
  - `httpPoller->indoorReceived` → `model->applyIndoorUpdate`
  - `udpReceiver->realtimeReceived` → `model->applyUdpUpdate`
- Connect thread lifecycle:
  - `networkThread->started` → `httpPoller->start`
  - `networkThread->started` → `udpReceiver->start`
  - `networkThread->finished` → `httpPoller->deleteLater`
  - `networkThread->finished` → `udpReceiver->deleteLater`
- Add a diagnostic connection for Phase 1 validation: connect `model->temperatureChanged` to a lambda that logs via `qDebug() << "Temperature:" << value;` (provides visible proof that data flows end-to-end). Similarly for sourceStaleChanged.
- Start the thread: `networkThread->start();`
- Handle clean shutdown: connect `app.aboutToQuit` to a lambda that calls `networkThread->quit()` then `networkThread->wait()`.
- `return app.exec();`

**Dependency injection per locked decision:** Model is instantiated in main and passed (via signal/slot connections) — no singleton, no global.

**Per locked decision:** The URL `weatherlinklive.local.cisien.com` is the mDNS hostname. This is a local-network-only application.
