3. **Switch bar coloring to use recentAvgSpeed**: In the barsCanvas `onPaint` handler (around line 126), change:
```qml
// OLD: var color = root.windSpeedColor(bin.avgSpeed)
// NEW: call parent dashboard's windSpeedColor with recentAvgSpeed

```

Since CompassRose is a child of the GridLayout which is a child of DashboardGrid (id: dashboard), the CompassRose needs a way to call DashboardGrid's windSpeedColor. The cleanest approach: add a `colorFunction` property to CompassRose that accepts a JS function reference:

```qml
// In CompassRose root:
property var windSpeedColorFn: function(mph) { return "#5CA85C" }
```

Then in DashboardGrid.qml CompassRose instantiation:
```qml
windSpeedColorFn: function(mph) { return dashboard.windSpeedColor(mph) }
```

And in barsCanvas onPaint:
```qml
var color = root.windSpeedColorFn(bin.recentAvgSpeed !== undefined ? bin.recentAvgSpeed : bin.avgSpeed)
```

This eliminates the duplicate function and ensures CompassRose always uses DashboardGrid's color logic.

4. **Conditional needle vs center dot (WIND-01)**: Modify the pointerWrapper section (lines 152-215).

Replace the current pointer Canvas content with conditional rendering based on `root.windSpeed`:

When `root.windSpeed > 0` (or more precisely, `root.windSpeed >= 0.1` to match the calm threshold): render the existing tapered needle as-is.

When `root.windSpeed < 0.1` (calm): render only a small center dot. The dot should be smaller than the minimum bar length (`outerR * 0.08`). Use approximately `outerR * 0.04` radius. Gold color (#C8A000) to match dashboard palette. No rotation animation needed when calm.

Implementation approach -- modify the pointerCanvas onPaint function to branch:

```qml
onPaint: {
    var ctx = getContext("2d")
    ctx.clearRect(0, 0, width, height)

    var cx = width / 2
    var cy = height / 2
    var outerR = Math.min(width, height) * 0.5 * 0.95

    if (root.windSpeed < 0.1) {
        // Calm: small center dot only
        ctx.beginPath()
        ctx.arc(cx, cy, outerR * 0.04, 0, 2 * Math.PI)
        ctx.fillStyle = "#C8A000"
        ctx.fill()
        return
    }

    // Non-calm: draw needle as before
    // ... existing needle code ...
}
```

Also: hide the `pointerWrapper` rotation animation when calm to avoid the dot spinning. The simplest way: the dot is drawn at center, so rotation has no visual effect. No change needed -- rotation of a centered dot is a no-op visually.

5. **Repaint pointerCanvas when windSpeed changes**: Add an `onWindSpeedChanged` handler or bind pointerCanvas repaint:

```qml
// In pointerCanvas or root:
Connections {

}
```

Or more simply, add to the pointerCanvas:
