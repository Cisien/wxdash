**5. Raspberry Pi Deployment section** — the primary deployment guide:

**5a. Prerequisites:**
- Raspberry Pi 4 or 5 running Pi OS Bookworm (64-bit recommended)
- Qt 6 development packages: `sudo apt install qt6-base-dev qt6-declarative-dev qt6-svg-dev libqt6quick6 libqt6qml6 cmake ninja-build`
- Mesa V3D driver (default on Pi OS Bookworm, no action needed)
- gpu_mem: default 76 MB is fine for Mesa; increase to 128 MB only if rendering issues (`/boot/firmware/config.txt`)

**5b. Build and Install:**
```bash
# Clone and build on the Pi
git clone <repo-url> wxdash
cd wxdash
cmake -B build -G Ninja -DCMAKE_BUILD_TYPE=Release -DCMAKE_INSTALL_PREFIX=/usr/local
cmake --build build

# Install kiosk files
sudo cmake --install build --component kiosk

# Or use convenience target (also needs sudo):
sudo cmake --build build --target install-kiosk
```

**5c. User Permissions:**
```bash
# Add user to required groups for DRM/input access
sudo usermod -aG render,video,input pi
# Log out and back in for group changes to take effect
```

**5d. EGLFS Configuration:**
- The installed eglfs.json at `/usr/local/share/wxdash/eglfs.json` targets `/dev/dri/card1` (correct for Pi 4/5 with Mesa V3D)
- If display is blank, check DRM device: `ls /dev/dri/` — on some Pi OS versions, the render node may be card0 instead of card1
- To override: edit `/usr/local/share/wxdash/eglfs.json` and change the `device` field
- Verify correct node: `sudo drminfo /dev/dri/card1` should show the V3D driver

**5e. systemd Service:**
```bash
# Enable auto-start at boot
sudo systemctl daemon-reload
sudo systemctl enable wxdash.service
sudo systemctl start wxdash.service

# Check status
systemctl status wxdash.service

# View logs
journalctl -u wxdash.service -f

# Stop/disable
sudo systemctl stop wxdash.service
sudo systemctl disable wxdash.service
```

**5f. Network / Hostname Resolution:**
- The app connects to `weatherlinklive.local.cisien.com` — this is a DNS name (NOT plain mDNS `.local`)
- Verify resolution on Pi: `nslookup weatherlinklive.local.cisien.com`
- If unresolved, add to `/etc/hosts`: `<IP_ADDRESS> weatherlinklive.local.cisien.com`
- PurpleAir sensor is accessed at `http://10.1.255.41/json` — verify Pi can reach this IP

**5g. Troubleshooting:**
- Black screen at boot → check `journalctl -u wxdash -b` for DRM/EGLFS errors; verify card device in eglfs.json
- Permission denied on /dev/dri → ensure user in render + video groups; check `groups pi`
- QML module not found → verify QML files exist near binary; set `QML_IMPORT_PATH` if needed
- No weather data → verify hostname resolution and network connectivity to WeatherLink Live
- High CPU / thermal throttle → ensure Release build (`CMAKE_BUILD_TYPE=Release`), not Debug
