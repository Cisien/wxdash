**Threshold color functions** — declare as JS functions at the top level of the component:

```javascript
function temperatureColor(tempF) {
    if (tempF < 32)  return "#5B8DD9"   // subdued blue
    if (tempF < 50)  return "#7BB8E8"   // subdued light blue
    if (tempF < 70)  return "#5CA85C"   // subdued green
    if (tempF < 85)  return "#C8A000"   // subdued yellow
    if (tempF < 100) return "#C87C2A"   // subdued orange
    return "#C84040"                    // subdued red
}

function humidityColor(hum) {
    if (hum < 25)  return "#C84040"     // subdued red (too dry)
    if (hum < 30)  return "#C87C2A"     // subdued orange
    if (hum < 60)  return "#5CA85C"     // subdued green (comfortable)
    if (hum < 70)  return "#C8A000"     // subdued yellow
    return "#C84040"                    // subdued red (too humid)
}

function windSpeedColor(mph) {
    if (mph < 5)   return "#5CA85C"     // subdued green
    if (mph < 15)  return "#C8A000"     // subdued yellow
    if (mph < 30)  return "#C87C2A"     // subdued orange
    return "#C84040"                    // subdued red
}

function uvColor(uv) {
    if (uv <= 2)  return "#5CA85C"      // subdued green
    if (uv <= 5)  return "#C8A000"      // subdued yellow
    if (uv <= 7)  return "#C87C2A"      // subdued orange
    if (uv <= 10) return "#C84040"      // subdued red
    return "#8B5CA8"                    // subdued violet
}
```

**Feels-like logic** — computed properties:

```javascript
// Feels-like value: heat index, wind chill, or raw temperature
property real feelsLikeValue: {
    if (weatherModel.temperature >= 80 && weatherModel.humidity >= 40)
        return weatherModel.heatIndex
    if (weatherModel.temperature <= 50 && weatherModel.windSpeed >= 3)
        return weatherModel.windChill
    return weatherModel.temperature
}

// Label showing which formula is active
property string feelsLikeLabel: {
    if (weatherModel.temperature >= 80 && weatherModel.humidity >= 40)
        return "Heat Index"
    if (weatherModel.temperature <= 50 && weatherModel.windSpeed >= 3)
        return "Wind Chill"
    return "Feels Like"
}
```

IMPORTANT: The above uses block syntax `{ ... }` with `return` statements inside `property` bindings. This is valid QML — a property binding can be a JavaScript block expression when it contains `if/return` statements.

**src/qml/CompassRose.qml — Traditional compass rose with smooth rotation:**

Create a Canvas-based compass rose component with the following structure:

```qml
import QtQuick
```

Top-level `Item` with:
- `property int windDir: 0` — bound to `weatherModel.windDir` by DashboardGrid

**Canvas element (the rose drawing):**
- `anchors.fill: parent` with margins to leave room for the direction label at bottom
- `anchors.bottomMargin: parent.height * 0.12` (space for direction label text)
- `rotation: -windDir` — negative because the rose rotates opposite to wind direction (wind coming FROM that direction, so the arrow points into the wind. Actually: compass convention is the rose shows where wind comes FROM, so the north pointer on the rose rotates to point to the wind's origin. With `rotation: -windDir`, when wind is from the East (90 deg), the north pointer rotates 90 degrees counterclockwise, putting the East label at the top/pointer position.)

CORRECTION: Standard wind vane convention — the arrow/pointer indicates wind direction (where wind is coming FROM). If windDir = 90 (East), the pointer should point East. The simplest approach: draw a fixed pointer at the top (North position), then rotate the entire rose by `-windDir` so that the compass direction matching windDir ends up at the top. When windDir=0 (North), rotation=0, N is at top. When windDir=90 (East), rotation=-90, E moves to top. This is correct.

- `Behavior on rotation` containing `RotationAnimation { duration: 400; direction: RotationAnimation.Shortest }` — handles 359-to-1 wraparound correctly (GAUG-05 requirement: update within 2.5s)

- `onPaint:` handler draws the compass rose using the Canvas 2D context:

Drawing instructions (all coordinates relative to canvas center, all sizes proportional to `Math.min(width, height) * 0.5` as the base radius):

1. Colors: Use `"#C8A000"` (subdued yellow) for all lines, text, and the arrow. Use `"#2A2A2A"` for the outer circle fill.

2. Outer circle: `ctx.arc(cx, cy, radius * 0.95, 0, 2*Math.PI)` — stroke with subdued yellow, thin line (1.5px).

3. Cardinal direction tick marks (N, E, S, W): Lines from `radius * 0.80` to `radius * 0.95` from center. Thicker line (2.5px).

4. Intercardinal tick marks (NE, SE, SW, NW): Lines from `radius * 0.85` to `radius * 0.95`. Medium line (1.5px).

5. Secondary intercardinal tick marks (NNE, ENE, ESE, SSE, SSW, WSW, WNW, NNW): Lines from `radius * 0.88` to `radius * 0.95`. Thin line (1px).

6. Cardinal labels: Draw "N", "E", "S", "W" text at `radius * 0.65` from center in their respective directions. Font: bold, `radius * 0.16` pixel size. Color: subdued yellow. Use `ctx.save()` / `ctx.translate()` / `ctx.rotate()` / `ctx.fillText()` / `ctx.restore()` pattern — each label is drawn upright (compensate for canvas rotation so text is always readable). NOTE: Since the canvas itself rotates, the text rotates with it. This is acceptable — it's a traditional compass rose where labels rotate with the instrument.

7. North pointer: An elongated triangle/diamond pointing from center toward the N direction. Fill with subdued yellow. The pointer extends from center to `radius * 0.75`. Width at base: `radius * 0.12`. This gives a clear directional indicator.

8. `onWidthChanged: requestPaint()` and `onHeightChanged: requestPaint()` — repaint only on resize, NOT on rotation changes. The rotation is handled by the scene graph transform, not by repainting. This is critical for performance (see research Pitfall 1).

**Direction label text (outside the Canvas):**

A `Text` element anchored at the bottom center of the parent Item:

```javascript
function directionLabel(deg) {
    var dirs = ["N","NNE","NE","ENE","E","ESE","SE","SSE",
                "S","SSW","SW","WSW","W","WNW","NW","NNW"]
    return dirs[Math.round(deg / 22.5) % 16]
}
```

This text does NOT rotate with the canvas — it's a separate Text element that always shows the current direction as readable text.
