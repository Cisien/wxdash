**src/qml/ArcGauge.qml — Reusable arc gauge component:**

A Shape-based arc gauge that all 9 gauge types will instantiate with different configurations. Uses `ShapePath` with `strokeColor` (ring style, not pie wedge fill) per research recommendation for fitness-tracker-ring aesthetic.

Properties to expose:
- `property real value: 0.0` — current gauge value (bound to weatherModel property)
- `property real minValue: 0.0` — range minimum
- `property real maxValue: 100.0` — range maximum
- `property string label: ""` — gauge title text (e.g., "Temperature")
- `property string unit: ""` — unit label (e.g., "°F", "mph")
- `property int decimals: 1` — decimal places for value display
- `property color arcColor: "#B8860B"` — arc fill color (dark yellow default; overridden by threshold gauges)
- `property string secondaryText: ""` — optional secondary line (gust readout, daily rain, trend arrow, etc.)
- `property string secondaryLabel: ""` — label for secondary display (e.g., "Gust", "Daily")

Implementation:
1. Arc geometry: `startAngle: 125`, `sweepAngle: 290` (290-degree sweep = ~80% of circle, leaving a 70-degree gap at the bottom). Start from bottom-left, sweep clockwise to bottom-right.
2. Track arc: `ShapePath` with `strokeColor: "#2A2A2A"` (dim muted track), `fillColor: "transparent"`, `strokeWidth: Math.min(width, height) * 0.08`, `capStyle: ShapePath.RoundCap`. Single `PathAngleArc` centered in the item.
3. Fill arc: `ShapePath` with `strokeColor: root.arcColor`, `fillColor: "transparent"`, same `strokeWidth` and `capStyle`. Declare `property real animatedSweep: 0` on the ShapePath. Use `Behavior on animatedSweep` containing `SmoothedAnimation { velocity: 200 }` (degrees/sec). Bind `animatedSweep` target to computed value sweep: `290 * Math.max(0, Math.min(1, (value - minValue) / (maxValue - minValue)))`. `PathAngleArc.sweepAngle` binds to `parent.animatedSweep`.

   CRITICAL: SmoothedAnimation on a property inside ShapePath — the `animatedSweep` property lives on the ShapePath, not on PathAngleArc. PathAngleArc.sweepAngle reads from `parent.animatedSweep`. This avoids Pitfall 4 from research.

4. Value text: `Text` anchored to center, displays `value.toFixed(decimals) + " " + unit`. Color: `"#C8A000"` (subdued yellow). Font size: `Math.min(width, height) * 0.18`. Bold.
5. Label text: `Text` above center (anchored to parent.horizontalCenter, positioned at ~35% from top). Displays `label`. Color: `"#C8A000"`. Font size: `Math.min(width, height) * 0.09`.
6. Secondary text (if non-empty): `Text` below the value text. Displays `secondaryLabel + ": " + secondaryText` (or just `secondaryText` if secondaryLabel is empty). Color: `"#C8A000"`. Font size: `Math.min(width, height) * 0.08`.
7. All font sizes use `font.pixelSize` bound to `Math.min(width, height) * factor` for responsive scaling (KIOSK-02). No hardcoded pixel sizes.

**src/qml/Main.qml — Root window:**

```qml
import QtQuick
import QtQuick.Layouts

Window {
    id: root
    width: 1280
    height: 720
    visible: true
    title: "wxdash"
    color: "#1A1A1A"

    // Kiosk mode from C++ context property
    visibility: kioskMode ? Window.FullScreen : Window.Windowed
    flags: kioskMode ? Qt.Window | Qt.FramelessWindowHint : Qt.Window

    // F11 toggles fullscreen at runtime (KIOSK-01)
    Item {
        anchors.fill: parent
        focus: true
        Keys.onPressed: (event) => {
            if (event.key === Qt.Key_F11) {
                if (root.visibility === Window.FullScreen) {
                    root.visibility = Window.Windowed
                    root.flags = Qt.Window
                } else {
                    root.visibility = Window.FullScreen
                    root.flags = Qt.Window | Qt.FramelessWindowHint
                }
            }
        }
    }

    // Placeholder content — Plan 02-02 replaces with DashboardGrid

        width: Math.min(parent.width, parent.height) * 0.4

        value: 72.5
        minValue: -20
        maxValue: 120
        label: "Temperature"
        unit: "\u00B0F"
        decimals: 1

}
```
